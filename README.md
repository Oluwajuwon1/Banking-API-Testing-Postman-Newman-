# Banking-API-Testing-Postman-Newman-
End-to-end API testing of a simulated banking system using Postman and Newman, covering login, balance validation, fund transfers, and automated HTML reporting.

Project Overview - This project demonstrates API testing for a simulated banking system using Postman and Newman. It covers key banking operations such as user login, balance validation, and fund transfers, with proper validations and error handling.
The goal of this project is to showcase practical QA engineering skills in API testing, including automation, dynamic data handling, and reporting.

Tools & Technologies - 
Postman (API Testing),
Newman (CLI Test Runner),
Newman HTML Extra Reporter,
MockAPI (Mock Backend - Mock.io),
JavaScript (Postman Scripts).

Features Covered
User Login Simulation (Dynamic & Manual),
Balance Validation before Transfer,
Fund Transfer Logic (Sender → Receiver),
Transaction Record Creation,
Error Handling (e.g., Insufficient Funds),
Dynamic Environment Variables,
Automated Test Execution using Newman,
HTML Test Reports Generation.

Why This Project Matters
This project reflects real-world fintech/banking API testing scenarios and demonstrates the ability to:
Think like a QA Engineer,
Validate business logic,
Automate API workflows,
Generate professional test reports,
